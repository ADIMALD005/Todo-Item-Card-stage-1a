This Stage 1 update builds on the original Stage 0 Todo Card by transforming it from a static display component into a more interactive, stateful, and accessible mini-experience while still remaining a single card and not a full application. The card now supports full editing through an edit mode with labeled form inputs for title, description, priority, and due date, along with proper save and cancel behaviors that restore or update state while returning focus correctly for better keyboard usability. Status is no longer just shown but can be actively changed through a control that stays synchronized with the checkbox and visual display, enforcing logical transitions between “Pending”, “In Progress”, and “Done”. The description area now supports expand and collapse behavior when content is long, implemented with accessible aria attributes and keyboard support. Priority is visually emphasized through a dynamic indicator that changes based on Low, Medium, or High values, while overdue tasks display a clear red accent and an explicit indicator. Time handling has been enhanced to update automatically every 30–60 seconds, showing granular messages such as “Due in 3 hours” or “Overdue by 45 minutes”, and stops updating once the task is marked as completed. The layout is responsive across mobile, tablet, and desktop, with careful handling of long titles, wrapped tags, and very large descriptions without breaking the design. Accessibility improvements include proper labeling, aria-live updates for time changes, logical tab flow, and improved keyboard interaction across all controls. Compared to Stage 0, this version focuses heavily on state synchronization, user interaction, and accessibility patterns rather than just visual structure. Design decisions prioritized clarity of state, visual feedback for priority and status, and smooth user flow between view and edit modes. A known limitation is that focus trapping inside the edit form is implemented only partially as a bonus enhancement. Overall, this version emphasizes clean state management, responsive layout behavior, and improved accessibility while keeping the component simple and modular.
